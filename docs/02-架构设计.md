# 架构设计
## 系统架构
```plaintext
┌─────────────────────────────────────────────────┐
│              浏览器 (localhost:5173)               │
│  ┌─────────────────────────────────────────────┐ │
│  │         Vue 3 前端 (Vite Dev Server)         │ │
│  │  App.vue → TodoList / TodoListApi / Monitor  │ │
│  └───────────┬──────────────┬─────────────────┘ │
└──────────────┼──────────────┼───────────────────┘
               │ REST API     │ WebSocket
               │ fetch()      │ new WebSocket()
┌──────────────┼──────────────┼───────────────────┐
│  ┌───────────┴──────────┐  │                     │
│  │ TS 后端 :3001        │  │                     │
│  │ Express + SQLite     │  │  ws://:3001/ws      │
│  └──────────────────────┘  │                     │
│  ┌──────────────────────┐  │                     │
│  │ Python 后端 :3002    │  │  ws://:3002/ws      │
│  │ Flask + SQLite       │  │                     │
│  └──────────────────────┘  │                     │
└──────────────────────────────────────────────────┘

```

## 前后端通信方式
### REST（HTTP 请求）— TodoListApi 组件使用
```plaintext
浏览器                               服务器
  │  GET /api/todos  ──────────→    │
  │  ←───── [{...}, {...}]  ──────  │
  │  POST /api/todos ──────────→    │
  │  ←───── {id: 1, ...}  ────────  │

```

- 前端主动发请求，后端被动响应
- 适合 CRUD 操作（增删改查）
- 使用浏览器内置的 fetch() 方法
### WebSocket（长连接）— Monitor 组件使用
```plaintext
浏览器                               服务器
  │  ──── 建立连接 ────────────→    │
  │  ←──── init (初始数据)  ──────  │
  │  ←──── health (每5秒)  ───────  │
  │  ←──── log (有请求时)  ────────  │

```

- 一次连接，后端可以主动推送数据
- 适合实时场景（监控、聊天、通知）
- 断线后自动重连（3 秒）
## 数据库设计
```sql
CREATE TABLE todos (
    id   INTEGER PRIMARY KEY AUTOINCREMENT,
    text TEXT NOT NULL,
    done INTEGER NOT NULL DEFAULT 0   -- SQLite 用 0/1 表示布尔值
);

```

每个后端维护独立的 todos.db 文件，数据互不共享。
## CORS 跨域配置
前端运行在 localhost:5173，后端在 localhost:3001/3002，端口不同即跨域。两个后端都配置了 CORS：
```typescript
// TS 后端 (index.ts)
app.use(cors({ origin: 'http://localhost:5173' }))

```

```python
# Python 后端 (app.py)
CORS(app, resources={r'/api/*': {'origins': 'http://localhost:5173'}})

```

## 设计原则
- **API 契约一致性**：两个后端实现完全相同的 RESTful API，体现"API 是语言无关的契约"
- **前后端分离**：前端通过 fetch/WebSocket 与后端通信，不直接访问数据库
- **教学导向**：代码注释详尽，每一步都有解释，适合初学者阅读
