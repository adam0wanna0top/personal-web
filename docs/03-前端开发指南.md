# 前端开发指南
## 技术栈
- Vue 3.5 + `<script setup lang="ts">` 语法糖
- TypeScript 5.7（严格模式）
- Vite 6 构建工具
- `<style scoped>` 组件级样式
## 组件架构
```plaintext
App.vue（根组件）
  ├── TodoList.vue       # 纯前端待办，无网络请求
  ├── TodoListApi.vue    # API 待办，通过 fetch() 连后端
  └── Monitor.vue        # WebSocket 实时监控面板

```

## App.vue — 根组件
展示 Vue 3 核心概念的入门示例：
- `ref()` — 响应式基本类型（title, name, count）
- `computed()` — 计算属性（parity：奇偶判断）
- `v-model` — 双向绑定（input 输入）
- `v-if / v-else` — 条件渲染
- `v-for` — 列表渲染（concepts 概念列表）
- `@click` — 事件绑定（计数器加减）
## TodoList.vue — 本地待办
纯前端实现，数据存在浏览器内存中：
- `ref<Todo[]>()` — 响应式数组
- `computed()` — doneCount 计算已完成数量
- `@submit.prevent` — 表单提交拦截
- `v-for` + `:key` — 列表渲染
- `v-model` — checkbox 双向绑定
- `:class="{ done: todo.done }"` — 动态 class 绑定
关键类型定义：
```typescript
interface Todo {
  id: number
  text: string
  done: boolean
}

```

## TodoListApi.vue — API 待办
通过 fetch() 与后端通信，演示完整的 CRUD：
- `onMounted(loadTodos)` — 组件挂载时自动加载数据
- `apiBase` ref — 可切换的 API 地址（TS:3001 / Python:3002）
- `@change="loadTodos"` — 切换后端时重新加载
- 错误处理 — try/catch + 回滚机制（toggleTodo 失败时恢复状态）
API 调用模式：
```typescript
// GET
const res = await fetch(`${apiBase.value}/todos`)
const data = await res.json()

// POST
await fetch(`${apiBase.value}/todos`, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ text }),
})

// PUT
await fetch(`${apiBase.value}/todos/${id}`, {
  method: 'PUT',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ done: true }),
})

// DELETE
await fetch(`${apiBase.value}/todos/${id}`, { method: 'DELETE' })

```

## Monitor.vue — 实时监控面板
通过 WebSocket 实时接收后端推送的监控数据：
- `new WebSocket(url)` — 建立 WebSocket 连接
- `ws.onmessage` — 接收三种消息：init / health / log
- 断线重连 — `ws.onclose` 中 setTimeout 3秒后重连
- `onUnmounted` — 组件卸载时关闭 WebSocket 连接
- 两个独立连接 — 同时连接 TS 和 Python 两个后端
消息类型：
```typescript
type: 'init'    // 初始连接时，后端一次性发送所有数据
type: 'health'  // 每 5 秒推送系统资源（内存、CPU、运行时间）
type: 'log'     // 每当有 API 请求时，立刻推送一条日志

```

## 测试
前端使用 Vitest + Vue Test Utils 进行单元测试。
### 测试框架
- **Vitest** — 与 Vite 集成的测试框架，配置在根目录 `vitest.config.ts`
- **Vue Test Utils** — Vue 官方组件测试工具（`@vue/test-utils`）
- **happy-dom** — 轻量 DOM 环境（替代 jsdom）
### 测试文件
```plaintext
frontend/__tests__/App.test.ts    # App.vue 组件测试

```

### 测试内容
- 渲染标题文字
- 输入框交互（v-model 双向绑定）
- 计数器按钮（递增、重置）
### 运行测试
```bash
npm test           # 单次运行
npm run test:watch # 监听模式

```

### 测试配置
`vitest.config.ts` 关键配置：
```typescript
export default defineConfig({
  plugins: [vue()],
  resolve: { alias: { '@': path.resolve(__dirname, './frontend') } },
  test: {
    environment: 'happy-dom',
    include: ['frontend/**/*.test.ts'],
  },
})

```
