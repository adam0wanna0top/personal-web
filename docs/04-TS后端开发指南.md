# TypeScript 后端开发指南
## 技术栈
- Express 4 + better-sqlite3 + ws（WebSocket）
- ESM 模式（package.json 中 `"type": "module"`）
- tsx 运行（不需要编译步骤）
## 项目结构
```plaintext
backend-ts/
├── package.json               # ESM 模式，tsx 运行
├── tsconfig.json
└── src/
    ├── index.ts               # Express 路由 + WebSocket 服务
    └── db.ts                  # SQLite 数据库 CRUD 操作

```

## 启动方式
```bash
cd backend-ts
npm install
npm run dev    # http://localhost:3001

```

## index.ts — 路由 + WebSocket
### 中间件配置
```typescript
app.use(cors({ origin: 'http://localhost:5173' }))  // CORS 跨域
app.use(express.json())                              // JSON body 解析

```

### 请求日志中间件
每个请求自动记录日志，并通过 WebSocket 广播给监控客户端：
```typescript
app.use((req, res, next) => {
  totalRequests++
  res.on('finish', () => {
    const log = { method, path, status, time }
    requestLogs.unshift(log)
    broadcast({ type: 'log', log, totalRequests })  // WebSocket 推送
  })
  next()
})

```

### WebSocket 实现
使用 `ws` 库，创建 WebSocketServer 挂在 HTTP server 上：
```typescript
const server = http.createServer(app)
const wss = new WebSocketServer({ server, path: '/ws' })

// 连接时发送初始数据
wss.on('connection', (ws) => {
  ws.send(JSON.stringify({ type: 'init', health, stats }))
})

// 每 5 秒推送系统资源
setInterval(() => {
  broadcast({ type: 'health', health: getHealthData() })
}, 5000)

```

### 系统监控
使用 Node.js 内置 `os` 模块获取系统信息：
```typescript
import os from 'os'

os.totalmem()    // 总内存
os.freemem()     // 空闲内存
os.cpus()        // CPU 信息
os.loadavg()     // 系统负载

```

## db.ts — 数据库层
使用 better-sqlite3（同步 API，性能好）：
```typescript
import Database from 'better-sqlite3'
const db = new Database('todos.db')
```

### 类型转换
SQLite 用 0/1 存布尔值，查询时转换：
```typescript
.map((row: any) => ({ ...row, done: Boolean(row.done) }))

```

写入时转换：
```typescript
.run(text, done ? 1 : 0)

```

## 测试
TS 后端使用 Vitest 进行单元测试。
### 测试框架
- **Vitest** — 与项目构建工具一致，配置在 `backend-ts/vitest.config.ts`
- 测试文件位于 `src/__tests__/` 目录
### 测试文件
```plaintext
backend-ts/src/__tests__/db.test.ts    # 数据库 CRUD 单元测试

```

### 运行测试
```bash
cd backend-ts
npm test           # 单次运行
npm run test:watch # 监听模式

```

### CI 集成
PR 到 master 时自动运行类型检查和测试：
```bash
npx tsc --noEmit   # 类型检查
npm test           # 单元测试

```
