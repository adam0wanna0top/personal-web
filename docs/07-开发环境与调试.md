# 开发环境与调试
## 环境要求
- Node.js >= 18
- Python >= 3.10
- npm >= 9
## 启动所有服务
### 前端（端口 5173）
```bash
npm install        # 安装依赖（只需一次）
npm run dev        # 启动开发服务器


```

访问 http://localhost:5173
### TypeScript 后端（端口 3001）
```bash
cd backend-ts
npm install        # 安装依赖（只需一次）
npm run dev        # 启动


```

WebSocket: ws://localhost:3001/ws
### Python 后端（端口 3002）
```bash
cd backend-py
python3 -m venv venv           # 创建虚拟环境（只需一次）
source venv/bin/activate       # 激活虚拟环境
pip install -r requirements.txt # 安装依赖（只需一次）
python app.py                   # 启动


```

WebSocket: ws://localhost:3002/ws
> 两个后端可以同时启动，前端通过下拉框自由切换。

## 常见问题
### 启动前端后页面空白或报错
确认已经运行 `npm install` 安装依赖。
### API 版待办事项显示"无法连接后端"
需要先启动至少一个后端（TS 或 Python），前端才能拿到数据。
### Python 后端启动报错 ModuleNotFoundError
确保已激活虚拟环境并安装依赖：
```bash
cd backend-py
source venv/bin/activate
pip install -r requirements.txt


```

### 切换后端后数据不一样
两个后端各自使用独立的 todos.db 文件，数据互不共享。
### 监控面板显示"离线"
对应的后端未启动。启动后前端会自动重连（3 秒后）。
## 开发流程
```plaintext
1. 修改 .vue 文件 → 保存 → 浏览器自动刷新（热更新）
2. 新建组件 → 在父组件中 import → 使用
3. 出错了 → 看浏览器控制台（F12）或终端的错误提示


```

## Wiki 同步流水线
PR 合并到 master 后自动触发 Wiki Sync 工作流，自动同步飞书知识库：
**工作原理：**
1. 安装 lark-cli 和 Claude Code CLI
1. Claude Code 分析代码变更
1. 生成或更新对应的飞书文档
1. 自动追加变更记录
**手动触发：** GitHub Actions → Wiki Sync → Run workflow
**依赖 Secret：**
- `ANTHROPIC_BASE_URL`: API 基础 URL
- `ANTHROPIC_API_KEY`: Claude API 密钥
- `LARK_APP_ID`: 飞书应用 ID
- `LARK_APP_SECRET`: 飞书应用密钥
